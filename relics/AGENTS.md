# relics 기능 에이전트 규칙

## 적용 범위와 필수 문서

이 규칙은 `relics/` 전체에 적용한다. 루트부터 대상까지 모든 `AGENTS.md`를 읽는다. 용어는 `GAME_TERMS.md`의 성장과 빌드 절, 수용 기준은 `GROUND_RULES.md`의 D·E·F를 사용한다. 계획·변경·QA에는 `WORKFLOW.md`를 적용한다.

## 책임

- 유물의 안정적인 식별자, 설명과 위치 기반 효과 데이터를 제공한다.
- 획득한 유물 인스턴스와 보유 상태를 관리한다.
- 보드가 제공한 슬롯에 대한 배치, 이동과 제거 유효성을 검증한다.
- 위치, 충돌과 발사 이벤트를 유물 발동으로 변환한다.
- 핀볼과 스코어 기능에 적용할 중립적인 효과 명령을 제공한다.

## 유물 분류 기준

- 유물은 보드에 배치되며 위치에 따라 공의 경로, 스코어 또는 생존 가능성이 달라져야 한다.
- 위치를 바꿔도 가치와 결과가 달라지지 않는 효과는 유물이 아니라 `rewards/`의 패시브 효과로 분류한다.
- 유물은 고득점, 보스 공격 또는 안전 경로 중 하나 이상의 명확한 배치 목적을 가져야 한다.
- 특정 유물이 모든 보드와 상황에서 항상 정답이 되지 않게 조건과 상충 관계를 제공한다.

## 소유하지 않는 것

| 대상 | 소유 기능 |
| --- | --- |
| 보드 형태와 배치 가능한 슬롯 정의 | `stages/` |
| 공, 충돌, 플리퍼와 스코어 원본 계산 | `pinball/` |
| 보상 카드, 가격, 구매와 패시브 효과 | `rewards/` |
| 웨이브·스테이지 상태와 리트라이 | `game_flow/` |
| 배치 화면, 이펙트와 사운드 표현 | `ui/`, `presentation/` |

## 의존성 방향

```text
rewards → relics
relics → shared
```

- 구매가 성공하면 `rewards/`가 공개 획득 명령으로 유물을 지급한다.
- `relics/`는 `stages/`의 슬롯과 `pinball/`의 충돌 정보를 중립 계약으로 받으며 두 기능의 내부 노드를 참조하지 않는다.
- 효과는 공개 명령 또는 이벤트로 전달하고 공, 스코어와 보스 값을 직접 변경하지 않는다.
- `relics/`는 `game_flow/`, `app/`, `ui/`, `presentation/`을 직접 참조하지 않는다.

## 상태와 미확정 정책

- 유물 인스턴스의 보유 상태와 배치 상태를 분리한다.
- 동일 인스턴스는 동시에 하나의 슬롯에만 배치할 수 있다.
- 유물 배치 시점은 `D-06`, 실패·리트라이 때 초기화 범위는 `D-03`, 스테이지 간 유지 범위는 `D-09` 결정 전까지 확정하지 않는다.
- 임시 수명 범위가 필요하면 외부에서 주입하는 정책으로 두고 승인된 작업 기록에 교체 지점을 남긴다.

## 필수 검증

- 동일 유물을 위치 A/B에 배치했을 때 경로와 결과 차이를 같은 조건에서 비교한다.
- 잘못된 슬롯, 중복 배치와 보유하지 않은 유물의 배치를 거부하는지 확인한다.
- 동일 충돌·발사 이벤트가 한 유물을 중복 발동시키지 않는지 확인한다.
- 효과 적용 전후의 공 반응, 스코어 또는 경로 차이를 수치와 로그로 확인한다.
- 유물을 제거하거나 이동했을 때 이전 슬롯의 효과가 남지 않는지 확인한다.
- 물리 효과는 터널링, 끼임, 무한 반복, 과속과 필승 조합을 검증한다.
