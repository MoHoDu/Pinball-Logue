# relic placement 에이전트 규칙

## 책임

- 보드가 제공한 슬롯 정보와 유물의 배치 조건을 비교한다.
- 유물 배치, 이동, 교체와 제거를 하나의 일관된 상태 변경으로 처리한다.
- 활성 배치 목록을 발동 기능에 제공한다.

## 규칙

- 보유한 유물 인스턴스만 배치할 수 있고 한 인스턴스는 슬롯 하나만 점유한다.
- 슬롯 존재 여부, 허용 조건과 점유 상태를 모두 확인한 뒤 상태를 변경한다.
- 이동이나 교체가 실패하면 원래 배치를 보존한다.
- 보드 좌표와 슬롯 정의를 직접 만들지 않고 중립 슬롯 계약으로 받는다.
- 배치 화면과 입력을 소유하지 않으며 검증 결과와 실패 이유만 제공한다.
- 배치 가능 시점과 강제 배치 여부는 `D-06` 결정 전까지 확정하지 않는다.

## 필수 검증

- 존재하지 않는 슬롯, 점유 슬롯, 비호환 슬롯과 미보유 유물을 거부하는지 확인한다.
- 동일 인스턴스 중복 배치와 동시 이동 요청에서 상태가 손상되지 않는지 확인한다.
- 이동·교체 실패 뒤 원래 배치와 효과가 유지되는지 확인한다.
- 제거 또는 이동 뒤 이전 슬롯의 발동 등록과 효과가 남지 않는지 확인한다.
