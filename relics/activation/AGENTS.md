# relic activation 에이전트 규칙

## 책임

- 활성 배치와 중립 충돌·발사 이벤트를 유물 발동 조건과 비교한다.
- 발동 횟수, 발사당 제한과 재사용 대기 상태를 관리한다.
- 성공한 발동을 효과 기능에 식별 가능한 명령으로 전달한다.

## 규칙

- 이벤트 식별자와 유물 인스턴스 식별자를 함께 사용해 중복 발동을 막는다.
- 유물이 현재 배치된 위치와 이벤트 위치가 조건을 만족할 때만 발동한다.
- 배치가 이동·제거되면 이전 위치의 발동 등록을 즉시 해제한다.
- 충돌과 물리 결과를 직접 계산하거나 공·스코어 값을 변경하지 않는다.
- 발동 횟수, 재사용 대기와 발사당 제한은 데이터로 설정하고 시간 단위를 명시한다.

## 필수 검증

- 동일 이벤트 재전송이 같은 유물을 두 번 발동시키지 않는지 확인한다.
- 조건 경계의 안·밖과 정확한 경계 위치에서 발동 결과를 확인한다.
- 재사용 대기 직전·경계·직후와 발사 전환 때 상태를 확인한다.
- 이동·제거 뒤 이전 위치 이벤트가 발동하지 않는지 확인한다.
