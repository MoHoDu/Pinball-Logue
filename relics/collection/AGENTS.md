# relic collection 에이전트 규칙

## 책임

- 획득한 유물의 고유 인스턴스와 카탈로그 식별자를 연결한다.
- 보유 수량, 배치 가능 상태와 획득 출처 식별자를 관리한다.
- 같은 구매·지급 결과가 중복 전달돼도 유물을 두 번 지급하지 않는다.

## 규칙

- 구매 가격과 구매 가능 여부는 판단하지 않고 성공한 획득 명령만 처리한다.
- 유물 정의 데이터와 플레이 중 인스턴스 상태를 분리한다.
- 배치 위치는 `placement/`가 소유하며 컬렉션은 배치 가능 여부를 위한 보유 사실만 제공한다.
- 리트라이와 스테이지 전환 때 무엇을 유지할지는 `D-03`, `D-09` 결정 전까지 외부 정책으로 받는다.
- 알 수 없는 카탈로그 식별자와 중복 획득 명령 식별자는 명시적으로 거부한다.

## 필수 검증

- 같은 지급 명령의 재전송이 수량과 인스턴스를 중복 생성하지 않는지 확인한다.
- 같은 종류의 유물을 여러 개 보유할 때 인스턴스를 서로 구분하는지 확인한다.
- 제거·초기화 후 배치 기능에 유효하지 않은 인스턴스가 남지 않는지 확인한다.
- 수명 정책 변경 전후의 보유 상태가 승인된 범위와 일치하는지 확인한다.
