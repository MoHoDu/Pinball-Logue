# relic catalog 에이전트 규칙

## 책임

- 유물의 안정적인 식별자, 표시 정보, 배치 조건과 효과 참조를 제공한다.
- 위치에 따른 가치 변화와 주요 배치 목적을 데이터로 설명한다.
- 유물 데이터와 효과·발동 참조의 완전성을 검증한다.

## 규칙

- 식별자는 표시 이름과 분리하고 저장 또는 보유 데이터에서 임의로 바꾸지 않는다.
- 각 유물은 고득점, 보스 공격 또는 안전 경로 중 하나 이상의 배치 목적을 명시한다.
- 위치와 무관하게 지속되는 효과만 가진 항목은 카탈로그에 추가하지 않고 패시브로 분류한다.
- 설명에 표시하는 수치와 실제 효과 설정은 같은 데이터 원본을 사용한다.
- 장면 경로보다 검증 가능한 리소스 또는 식별자 참조를 우선한다.

## 필수 검증

- 식별자 중복, 필수 필드와 효과 참조 누락을 로드 전에 발견하는지 확인한다.
- 표시 설명의 조건·수치가 실제 효과 데이터와 일치하는지 확인한다.
- 위치 A/B 비교에서 결과 차이를 만들 수 없는 유물이 포함되지 않았는지 확인한다.
- 삭제나 이름 변경으로 보유·배치 데이터의 참조가 끊기지 않는지 확인한다.
