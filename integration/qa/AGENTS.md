# professional qa 에이전트 규칙

1. 루트와 대상 기능 규칙, SPEC, 활성 PLAN과 QA 인수 목록을 읽는다.
2. 환경·빌드·기준 커밋, 입력, 시드와 증거 저장 경로를 확인한다.
3. 정상→실패→경계→회귀 순으로 독립 실행하고 `통과`, `수정`, `재검증`으로 기록한다.
4. 결과마다 실제값, 기대값, 횟수, 로그와 가능한 캡처·녹화를 남긴다.
5. 버그 발견 시 세션으로 사용자에게 먼저 보고하고 Jira `버그` 생성 여부를 묻는다.
6. 승인된 경우에만 Jira 이슈를 만들고 버그를 PLAN과 QA 재검증 항목에 추가한다.
7. 일자별 Markdown을 완성한 뒤 Confluence 게시 여부를 묻고 승인된 링크에만 반영한다.

전문 QA는 구현자의 자체 검증을 그대로 통과로 복사하지 않고 수용 기준에서 다시 판단한다.
