# pinball 기능 에이전트 규칙

## 적용 범위와 필수 문서

이 규칙은 `pinball/` 전체에 적용한다. 루트부터 대상까지 모든 `AGENTS.md`를 읽는다. 용어는 `GAME_TERMS.md`의 핀볼 플레이·조작과 점수·충돌 절을, 수용 기준은 `GROUND_RULES.md`의 A·C·F·G를 사용한다. 계획·변경·QA에는 `WORKFLOW.md`를 적용한다.

## 책임

- 단일 공의 물리 이동과 추적 가능한 상태를 관리한다.
- 발사 준비와 실행, 플리퍼 선택·작동 및 공 패링을 구현한다.
- 범퍼, 벽, 페그, 드레인 등 재사용 가능한 물리 오브젝트를 제공한다.
- 한 발사의 시작과 종료, 충돌·스코어 결과를 화면과 무관한 계약으로 제공한다.
- 충돌 점수, 콤보 시간과 스코어 배수를 계산한다.

## 소유하지 않는 것

| 대상 | 소유 기능 |
| --- | --- |

| 유물 정의, 배치와 효과 정책 | `relics/` |

| UI, 카메라, 이펙트와 사운드 표현 | `ui/`, `presentation/` |

## 의존성 방향

```text
app → pinball
stages → pinball
pinball → shared
```

- `stages/`는 공개 설정과 재사용 장면으로 보드를 구성하며, `pinball/`은 특정 스테이지를 알지 않는다.
- `pinball/`은 `game_flow/`, `relics/`, `rewards/`, `ui/`, `presentation/`을 직접 참조하지 않는다.
- 충돌, 점수, 수집과 발사 종료는 중립 이벤트 또는 `shared/` 계약으로 전달한다.
- 외부 기능이 물리 값을 바꿔야 하면 승인된 설정·효과 인터페이스를 사용하고 내부 노드를 직접 조작하지 않는다.

## 불변 규칙

- 보드에는 활성 공이 동시에 하나만 존재한다. 멀티볼과 공 분열은 구현하지 않는다.
- 입력은 `상황 확인 → 경로 선택 → 플리퍼 선택 → 타이밍 입력` 경험을 지원해야 한다.
- 결과에 영향을 주는 물리·점수 값은 설정 가능하고 반복 테스트할 수 있어야 한다.
- 같은 충돌이나 종료 신호를 중복 처리해 점수 또는 진행 결과가 두 번 발생하지 않아야 한다.
- 공 성장 후에도 위치와 이동 경로를 추적할 수 있어야 한다.
- 물리 변경은 터널링, 끼임, 무한 반복, 과속, 플리퍼 관통과 필승 조합을 검증한다.

## 미확정 정책 처리

낙하는 발사의 종료일 뿐 자동으로 스테이지 실패를 뜻하지 않는다. 공 라이프와 실패 계산은 `D-01`, `D-02` 결정 전까지 `pinball/`에서 확정하지 않는다. 공격력 패시브의 적용 대상은 `D-07` 결정 전까지 스코어, 충돌력 또는 보스 피해 중 하나로 임의 고정하지 않는다.

## 필수 검증

- 활성 공이 항상 0개 또는 1개인지 확인한다.
- 발사 하나가 정확히 한 번 시작하고 한 번 종료되는지 확인한다.
- 플리퍼 선택과 타이밍 차이가 공의 각도·거리 또는 속도에 측정 가능한 차이를 만드는지 확인한다.
- 동일 입력·설정·시드의 반복 결과와 물리 변경 전후를 비교한다.
- 충돌, 콤보, 배수와 최종 스코어의 계산 근거를 로그로 재현할 수 있는지 확인한다.
- 스코어와 코인 이벤트가 서로 다른 계약으로 전달되는지 확인한다.
