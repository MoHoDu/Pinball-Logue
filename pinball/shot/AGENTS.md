# shot 기능 에이전트 규칙

## 책임

- 한 발사의 준비, 진행과 종료 상태를 관리한다.
- 발사 동안 발생한 충돌, 획득과 스코어 결과의 식별 가능한 요약을 만든다.
- 낙하 또는 상위 기능이 제공한 종료 조건 중 먼저 확정된 하나로 발사를 종료한다.

## 규칙

- 개념적 상태는 `준비 → 진행 중 → 종료` 순서로만 전환한다.
- 발사 식별자를 사용해 이전 발사의 지연 이벤트와 현재 결과를 구분한다.
- 종료 이유와 최종 요약은 한 번만 확정하며 이후 이벤트는 결과를 변경하지 않는다.
- 낙하는 발사 종료 이유다. 공 라이프 소비와 스테이지 실패 여부는 결정하지 않는다.
- `game_flow/`를 직접 호출하지 않고 중립 결과를 조립 계층에 제공한다.

## 필수 검증

- 동시 또는 중복 시작 요청에도 활성 발사가 하나인지 확인한다.
- 낙하와 목표 달성 신호가 같은 물리 틱에 올 때 종료 결과가 결정적으로 하나인지 확인한다.
- 종료 후 지연 충돌이 최종 스코어와 진행을 바꾸지 않는지 확인한다.
- 결과 요약의 충돌·스코어·수집 합계가 원본 이벤트 로그와 일치하는지 확인한다.
