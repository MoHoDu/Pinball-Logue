# scoring 기능 에이전트 규칙

## 책임

- 유효한 충돌 이벤트를 기본 스코어로 변환한다.
- 콤보 시간 안의 연속 타격, 콤보 단계와 스코어 배수를 계산한다.
- 설정으로 받은 목표 스코어의 도달 사실을 한 번만 알린다.
- 각 점수 증가의 원인, 기본값, 배수와 최종값을 진단 이벤트로 제공한다.

## 계산 규칙

- 계산 순서와 반올림 규칙은 코드와 테스트에서 하나로 고정하고 변경 시 승인된 기획 문서에 기록한다.
- 동일한 충돌 식별자를 중복 반영하지 않는다.
- 콤보 유지·종료는 물리 프레임 수가 아니라 명시된 시간 기준으로 계산한다.
- 목표 스코어 값은 `stages/`에서 설정으로 받고 특정 스테이지 값을 하드코딩하지 않는다.
- 스코어와 코인은 별도 값과 이벤트로 유지한다.
- UI, 사운드와 이펙트는 계산 결과를 구독하며 스코어 원본을 변경하지 않는다.
- 공격력 패시브는 `D-07` 결정 전까지 자동으로 스코어 배수에 포함하지 않는다.

## 필수 검증

- 기본 스코어, 배수와 최종 증가량이 명시된 계산식과 일치하는지 확인한다.
- 콤보 시간의 직전·경계·직후 입력에서 유지와 종료가 정확한지 확인한다.
- 중복 충돌이 점수를 두 번 올리지 않는지 확인한다.
- 목표 스코어를 한 번에 초과해도 도달 이벤트가 한 번만 발생하는지 확인한다.
- 첫 10초의 충돌·콤보·스코어 변화 원인을 로그와 플레이 화면에서 추적 가능한지 확인한다.
