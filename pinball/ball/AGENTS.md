# ball 기능 에이전트 규칙

## 책임

- 공의 생성, 활성화, 물리 속성 적용과 안전한 제거를 관리한다.
- 위치, 속도와 활성 상태를 발사 및 충돌 기능에 제공한다.
- 승인된 효과 입력을 검증한 뒤 공의 물리 설정에 반영한다.

## 규칙

- 활성 공을 새로 만들기 전에 기존 활성 공의 존재를 확인한다.
- 공은 웨이브, 스테이지, 유물, UI와 화면 전환을 알지 않는다.
- 외부 기능은 공 내부 노드를 직접 변경하지 않고 공개 설정 또는 효과 계약을 사용한다.
- 질량, 반발, 감쇠, 중력 영향과 최대 속도는 설정값으로 노출하고 유효 범위를 검증한다.
- 제거는 발사 종료 결과가 확정된 뒤 한 번만 수행하며, 제거 자체로 실패를 선언하지 않는다.

## 필수 검증

- 중복 생성 요청에도 활성 공이 하나를 넘지 않는지 확인한다.
- 최소·최대 설정값에서 NaN, 무한 속도와 비정상 정지가 없는지 확인한다.
- 고속 충돌의 터널링, 오브젝트 끼임과 보드 밖 이탈을 확인한다.
- 성장 전후에도 공의 위치와 경로가 시각적으로 추적 가능한지 확인한다.
