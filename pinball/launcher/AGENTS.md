# launcher 기능 에이전트 규칙

## 책임

- 발사 가능한 공을 준비하고 방향과 세기 입력을 유효 범위로 변환한다.
- 승인된 입력 한 번에 공 하나를 한 번만 발사한다.
- 실제 적용된 초기 위치, 방향, 세기와 속도를 진단 가능한 결과로 제공한다.

## 규칙

- 개념적 상태는 `비활성 → 준비 → 발사 완료` 순서로만 전환한다.
- 활성 발사가 끝나기 전에는 새 발사를 허용하지 않는다.
- 방향과 세기의 기본값·범위는 설정 데이터로 받고 특정 보드 값을 하드코딩하지 않는다.
- 입력 장치와 UI 표현을 직접 소유하지 않고 정규화된 발사 명령만 받는다.
- 발사기는 공 라이프를 소비하거나 웨이브 실패를 판단하지 않는다.

## 필수 검증

- 중복 입력으로 공이 두 번 생성되거나 가속되지 않는지 확인한다.
- 최소·최대 세기와 경계 방향이 설정 범위를 벗어나지 않는지 확인한다.
- 실제 초기 속도가 요청한 방향·세기와 일치하는지 로그로 확인한다.
- 같은 설정의 반복 발사에서 의도하지 않은 초기 상태 차이가 없는지 확인한다.
