# reward purchases 에이전트 규칙

- 활성 제안, 카드 미구매 상태, 가격, 잔액과 외부 구매 정책을 모두 검증한다.
- 구매 식별자로 재시도를 멱등 처리하고 동일 요청을 두 번 차감·지급하지 않는다.
- 코인 차감과 유물·패시브 지급은 함께 커밋하거나 모두 되돌린다.
- 카드 하나의 제안 인스턴스는 한 번만 구매하며 총 구매 상한은 `D-05` 결정 전까지 고정하지 않는다.
- 성공과 실패 이유를 UI 및 QA가 사용할 중립 결과로 제공한다.
- 잔액 부족, 지급 실패, 동시 구매와 재전송 시 원자성을 검증한다.
