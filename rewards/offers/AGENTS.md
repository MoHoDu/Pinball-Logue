# reward offers 에이전트 규칙

- 유효한 카드 풀에서 정확히 3개를 제안하고 웨이브마다 새 제안 식별자를 만든다.
- 시드, 후보 풀 버전, 필터와 최종 결과를 기록해 같은 제안을 재현할 수 있게 한다.
- 카드 풀이 3개보다 작거나 참조가 잘못되면 임의 중복 대신 명시적으로 실패한다.
- 단일 제안 안의 같은 카드 중복 여부는 승인된 기획 데이터로 제어한다.
- 제안 생성은 코인을 차감하거나 구매·진행 상태를 바꾸지 않는다.
- 동일 시드 재현, 다른 시드 분포와 전략 방향 편중을 검증한다.
