# reward passives 에이전트 규칙

- 위치와 무관하게 지속되는 효과만 소유하며 위치가 중요하면 `relics/`로 분류한다.
- 정의 데이터, 획득 인스턴스와 현재 적용 상태를 분리한다.
- 효과는 공개 계약으로 적용하고 공과 스코어 내부 노드를 직접 변경하지 않는다.
- 중첩, 상한, 순서, 단위와 지속 범위를 명시하고 중복 지급을 막는다.
- 최소한 현재 스테이지 보스까지 유지하되 리트라이와 다음 스테이지 범위는 `D-03`, `D-09`를 따른다.
- 공격력 패시브 대상은 `D-07` 결정 전까지 확정하지 않는다.
- 적용 전후의 수치·공 반응·경로 변화를 같은 조건에서 비교한다.
