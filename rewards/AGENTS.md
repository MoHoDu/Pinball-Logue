# rewards 기능 에이전트 규칙

## 책임과 구조

`GAME_TERMS.md`의 보상·자원 용어와 `GROUND_RULES.md`의 D·F를 적용한다.

| 경로 | 책임 |
| --- | --- |
| `cards/` | 유물 종류·수량, 패시브와 가격을 묶은 카드 데이터 |
| `offers/` | 일반 웨이브마다 랜덤 카드 3개 제안 |
| `economy/` | 코인 획득, 잔액, 가격과 경제 측정 |
| `purchases/` | 구매 검증, 차감과 지급의 원자적 처리 |
| `passives/` | 위치와 무관하게 지속되는 공·스코어 효과 |
| `.work/` | 승인된 계획, QA와 증거 |

## 경계와 의존성

```text
game_flow → rewards
rewards → relics
rewards → shared
```

- 코인 배치와 접촉 감지는 `stages/`, `pinball/`이 담당하고 `rewards/`는 검증된 획득 결과를 잔액에 반영한다.
- 유물 지급은 `relics/`의 공개 획득 계약을 사용한다.
- 패시브는 `pinball/` 내부 노드를 직접 바꾸지 않고 중립 효과 계약을 제공한다.
- 화면 표시와 전환은 `ui/`, `app/`, `game_flow/`가 담당한다.

## 불변 규칙

- 일반 웨이브마다 카드 3개를 새로 제안하고 생성 시드와 결과를 기록한다.
- 무료 1개 선택이 아니라 보유 코인 범위에서 서로 다른 카드를 복수 구매한다.
- 모든 카드는 유물 종류·수량, 패시브 효과와 가격을 비교 가능한 형태로 제공한다.
- 스코어와 코인은 별도 값, 이벤트와 표시 계약을 사용한다.
- 구매는 검증·차감·보상 지급이 함께 성공하거나 모두 취소돼야 한다.
- 평균 코인 획득량과 안정 보상 가격은 매 화면에서 안정 보상 2개 구매 가능성을 함께 측정한다.

## 미확정 정책

리트라이 초기화는 `D-03`, 코인 유지 범위는 `D-04`, 구매 개수 상한은 `D-05`, 공격력 패시브 대상은 `D-07`, 스테이지 간 유지 범위는 `D-09` 결정 전까지 확정하지 않는다. 카드 하나의 동일 제안 인스턴스는 한 번만 구매하되 서로 다른 카드의 총 구매 상한은 외부 정책으로 받는다.

## 필수 검증

- 제안마다 정확히 카드 3개가 있고 같은 시드로 재현되는지 확인한다.
- 잔액 부족, 중복 요청과 이미 구매한 제안을 거부하는지 확인한다.
- 지급 실패 때 코인이 차감되지 않고 차감 실패 때 보상이 지급되지 않는지 확인한다.
- 구매 직후 잔액, 카드 상태와 획득 보상이 함께 갱신되는지 확인한다.
- 코인 획득량, 안정 보상 가격과 2개 구매 가능 비율을 같은 표본에서 측정한다.
