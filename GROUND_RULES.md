# Pinball_Logue 그라운드룰

이 문서는 구현과 QA의 로컬 수용 기준이다. 작업자는 관련 규칙 ID를 작업 계획과 검증 결과에 기록한다.

## A. 핵심 경험

| ID | 그라운드룰 | 적용·검증 기준 |
| --- | --- | --- |
| GR-01 | 최우선 재미는 공의 연쇄 충돌과 콤보로 스코어가 폭발적으로 상승하는 경험이다. | 첫 10초 안에 강한 충돌 또는 연속 타격을 경험하고, 한 발사에서 스코어 상승 원인을 읽을 수 있어야 한다. |
| GR-02 | 한 발사의 핵심 판단은 플리퍼 타이밍 자체가 아니라 목표 경로 선택이다. | 플레이어가 타격 전에 안전·고득점·보스 공격 중 목표를 설명할 수 있어야 한다. |
| GR-03 | 플리퍼 타이밍은 선택한 경로를 실행하고 액션·타격감을 발생시키는 수단이다. | 입력 타이밍 차이가 공의 도달 거리, 각도와 타격 피드백을 바꿔야 한다. |
| GR-04 | 판단 순서는 상황 확인 → 목표 경로 선택 → 플리퍼 선택 → 타이밍 입력이다. | 실패 후 어느 단계의 판단을 개선할지 설명할 수 있어야 한다. |
| GR-05 | 공은 완전히 통제할 수 없지만 플레이어의 선택이 결과 확률을 유의미하게 바꿔야 한다. | 의도한 경로의 성공률이 무작위 입력보다 높고 항상 성공하지는 않아야 한다. |
| GR-06 | 운적인 요소는 플레이어를 무력하게 만드는 실패보다 예상 이상의 좋은 결과를 만드는 데 우선 사용한다. | 예고 없는 손실보다 우연한 연쇄 충돌과 보너스 스코어를 더 자주 제공한다. |

## B. 스테이지와 웨이브

| ID | 그라운드룰 | 적용·검증 기준 |
| --- | --- | --- |
| GR-07 | 런은 여러 스테이지로 구성하며 현재 메인 로비에는 스테이지 1·2·3이 존재한다. | 현재 진행 위치와 다음 진입 가능 스테이지를 메인 로비에서 확인할 수 있어야 한다. |
| GR-08 | 한 스테이지는 일반 웨이브 3개와 보스 웨이브 1개로 구성한다. | 일반 1 → 보상 → 일반 2 → 보상 → 일반 3 → 보상 → 보스를 기본 흐름으로 사용한다. |
| GR-09 | 일반 웨이브를 클리어할 때마다 보상 선택 화면으로 이동한다. | 목표 스코어 달성 전에는 보상 화면으로 이동하지 않는다. |
| GR-10 | 세 번째 일반 웨이브의 보상까지 적용한 뒤 보스 웨이브에 진입한다. | 보스전 시작 전에 현재 빌드가 반영돼 있어야 한다. |
| GR-11 | 스테이지 내 어느 웨이브에서 실패하더라도 현재 스테이지의 일반 웨이브 1부터 리트라이한다. | 이전 스테이지 진행은 유지하며 현재 스테이지 내부 진행만 초기화한다. |
| GR-12 | 다음 웨이브와 다음 스테이지를 구분한다. | 일반 웨이브 클리어 뒤에는 다음 웨이브, 보스 클리어 뒤에는 다음 스테이지로 이동한다. |
| GR-13 | 보스 웨이브는 앞선 일반 웨이브에서 획득한 빌드와 플레이 숙련을 검증하는 최종 시험이다. | 일반 웨이브에서 익힌 플리퍼 선택과 공 패링이 보스전에서도 유효해야 한다. |

## C. 일반 웨이브

| ID | 그라운드룰 | 적용·검증 기준 |
| --- | --- | --- |
| GR-14 | 일반 웨이브의 핵심 목표는 스코어를 획득해 목표값을 달성하는 것이다. | 현재 스코어와 남은 목표값을 플레이 중 계속 표시한다. |
| GR-15 | 특정 작은 적이나 오브젝트를 맞히지 못해 진행이 정체되는 구조로 만들지 않는다. | 대부분의 충돌이 목표 진행에 일정 부분 기여해야 한다. |
| GR-16 | 범퍼, 콤보와 웨이브 아이템 등 주요 충돌이 스코어 진행에 기여해야 한다. | 무의미한 충돌 비율이 높으면 범퍼 기능과 배치를 조정한다. |
| GR-17 | 범퍼를 콤보 시간 안에 연속 타격하면 스코어 배수가 상승한다. | 콤보 유지 조건, 배수 상승과 종료가 시각·사운드로 명확해야 한다. |
| GR-18 | 웨이브 아이템은 보드 안에서 획득하며 스코어 배수 증가 등 즉시 이로운 효과를 준다. | 획득 즉시 효과와 지속 시간을 이해할 수 있어야 한다. |
| GR-19 | 일반 몬스터는 현재 프로토타입에 사용하지 않는다. | 일반 웨이브는 스코어링, 충돌과 공 패링 검증에 집중한다. |
| GR-20 | 코인은 핀볼 보드에서 직접 획득 가능해야 하며 스코어와 명확히 구분한다. | 아이콘, 색, 획득음과 UI 위치를 스코어와 다르게 설계한다. |

## D. 보상과 코인

| ID | 그라운드룰 | 적용·검증 기준 |
| --- | --- | --- |
| GR-21 | 각 일반 웨이브 클리어 후 랜덤 보상 카드 3개를 제시한다. | 웨이브마다 카드 구성이 갱신돼야 한다. |
| GR-22 | 보상 카드는 무료 선택지가 아니라 코인을 소비해 구매하는 상품이다. | 선택 전 보유 코인과 카드 가격을 동시에 확인할 수 있어야 한다. |
| GR-23 | 각 보상 카드에는 유물 종류와 수, 패시브 효과와 가격을 표시한다. | 한 카드의 내용을 한 화면에서 비교할 수 있어야 한다. |
| GR-24 | 플레이어는 보유 코인 범위에서 보상 카드를 복수 구매할 수 있다. | 구매 후 남은 코인과 구매 가능 카드가 즉시 갱신돼야 한다. |
| GR-25 | 코인은 매 보상 화면에서 최소 두 개의 안정 보상을 구매할 수 있을 정도로 획득 가능하게 배치한다. | 평균 코인 획득량과 안정 보상 가격을 함께 테스트한다. |
| GR-26 | 보상 카드 3개를 단순한 수치 상·중·하 차이로 구성하지 않는다. | 유물 배치, 공 패시브, 안전성, 고득점 등 서로 다른 방향을 제시한다. |
| GR-27 | 구매한 보상의 변화는 다음 웨이브에서 바로 확인할 수 있어야 한다. | 공의 반응, 스코어, 경로 또는 유물 발동 중 하나 이상이 달라져야 한다. |
| GR-28 | 구매한 보상은 최소한 해당 스테이지의 보스 웨이브까지 유지한다. | 웨이브 전환 시 구매 효과가 사라지지 않아야 한다. |

| --- | --- | --- |

| GR-30 | 같은 유물이라도 배치 위치에 따라 공의 경로와 결과가 달라져야 한다. | 동일 유물 위치 A/B 비교 테스트를 실시한다. |

| GR-32 | 유물 배치는 고득점, 보스 공격 또는 안전 경로 중 하나 이상의 목적을 가져야 한다. | 배치 이유가 `그냥 빈칸`이 되지 않도록 슬롯과 경로를 설계한다. |
| GR-33 | 특정 유물이 모든 보드와 모든 상황에서 항상 정답이 되어서는 안 된다. | 보드별 선택률과 승률이 과도하게 편중되면 조건과 성능을 조정한다. |
| GR-34 | `덱 빌딩` 대신 `보드 빌딩` 또는 `유물 배치 빌드`를 사용한다. | 기획서, Jira, Figma와 발표 자료에서 같은 용어를 사용한다. |
| GR-35 | 유물 획득과 배치 단계를 화면 흐름에서 생략하지 않는다. | 보상 구매 → 유물 확인 → 유물 배치/재배치 → 다음 웨이브 흐름을 표시한다. |

| GR-51 | 한 번의 작은 실수로 계속 실패시키지 않고 간헐적인 실패를 통해 학습하게 한다. | 공 라이프와 보드 위험도를 조절해 실패 빈도를 테스트한다. |

## 프로토타입 즉시 검증 체크리스트

각 항목은 `통과`, `수정`, `재검증` 중 하나로 기록하고 가능하면 수치, 로그, 캡처 또는 녹화를 첨부한다.

- [ ] 첫 플레이 10초 안에 연쇄 충돌 또는 강한 타격을 경험하는가?
- [ ] 플레이어가 안전·고득점·보스 공격 경로 중 목표를 말할 수 있는가?
- [ ] 플리퍼 선택과 타이밍 차이가 공의 경로와 타격감에 영향을 주는가?
- [ ] 범퍼 충돌, 콤보와 배수의 인과관계를 플레이어가 읽을 수 있는가?
- [ ] 코인과 스코어를 화면과 사운드에서 혼동하지 않는가?
- [ ] 카드 3개의 효과와 가격을 빠르게 비교하고 복수 구매할 수 있는가?
- [ ] 동일 유물의 위치를 바꾸면 실제 경로와 결과가 달라지는가?
- [ ] 구매 후 다음 웨이브에서 공이 강해졌다는 변화를 즉시 느끼는가?
- [ ] 일반 웨이브에서 익힌 조작과 빌드가 보스전에서 유효한가?
- [ ] 실패 원인을 개선 가능한 선택으로 인식하고 바로 리트라이하려는가?

## 미확정 결정 사항

다음 항목은 확정된 그라운드룰이 아니다. 작업자는 임의의 값을 영구 정책으로 만들지 않고, 구현상 임시값이 필요하면 사용자 승인과 변경 가능성을 기록한다.

| ID | 결정 항목 | 결정해야 할 질문 | 영향 범위 |
| --- | --- | --- | --- |
| D-01 | 스테이지 실패 조건 | 공 1회 낙하, 공 라이프 전체 소진, 별도 HP 소진 중 어떤 조건을 사용할 것인가? | 공 라이프, 난이도, 실패 빈도, 결과 화면 |
| D-02 | 공 라이프 수와 적용 단위 | 기본 수량과 웨이브·스테이지·런 중 어느 단위로 유지할 것인가? | 난이도, 리트라이 주기, UI |
| D-03 | 리트라이 초기화 범위 | 코인, 구매 보상, 유물 배치와 패시브를 어디까지 초기화할 것인가? | 성장 체감, 실패 부담, 세이브 로직 |
| D-04 | 코인의 유지 범위 | 코인이 웨이브 간, 스테이지 간 또는 런 전체에서 유지되는가? | 경제 밸런스, 가격, UI |
| D-05 | 보상 구매 개수 상한 | 코인이 충분하면 카드 3장을 모두 살 수 있는가, 구매 수를 제한할 것인가? | 선택의 긴장, 경제, 밸런스 |
| D-06 | 유물 배치 시점 | 구매 직후 강제 배치, 다음 웨이브 전 준비 화면, 언제든 재배치 중 무엇을 사용할 것인가? | 코어 루프, UI 흐름, 조작 복잡도 |
| D-07 | 공격력 패시브 적용 대상 | 일반 몬스터가 없을 때 보스 피해, 스코어, 충돌력 중 어디에 적용하는가? | 보상 설명, 일반 웨이브 효용, 보스 밸런스 |
| D-08 | 보스 클리어 보상과 다음 진행 | 영구 해금, 다음 스테이지 개방, 별도 보상 구매 중 무엇을 제공하는가? | 런 구조, 장기 성장, 결과 화면 |
| D-09 | 스테이지 간 유지 성장 | 현재 스테이지의 보상과 유물이 다음 스테이지까지 유지되는가? | 런 빌드, 난이도 곡선, 재도전 동기 |

결정이 완료되면 이 표에 결정일, 결정 내용, 승인자와 적용 버전을 추가하고 관련 용어·그라운드룰·코드·QA를 함께 갱신한다.
