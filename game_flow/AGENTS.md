# game_flow 기능 에이전트 규칙

## 적용 범위와 필수 문서

이 규칙은 `game_flow/` 전체에 적용한다. 루트 `AGENTS.md`를 항상 먼저 읽고, 대상 하위 디렉토리의 `AGENTS.md`를 이어서 읽는다. 계획·변경·QA에는 `WORKFLOW.md`, 진행 용어에는 `GAME_TERMS.md`, 수용 기준에는 `GROUND_RULES.md`의 B·D·H를 사용한다.

## 책임

- 현재 런, 스테이지와 웨이브 상태를 하나의 일관된 진행 모델로 관리한다.
- 허용된 상태 전환과 일반 웨이브·보상·보스 순서를 보장한다.
- 웨이브 및 스테이지의 클리어·실패 결과와 리트라이 진입을 처리한다.
- 다음 단계가 필요한 기능에 화면과 무관한 도메인 결과를 제공한다.

## 소유하지 않는 것

| 대상 | 소유 기능 |
| --- | --- |
| 시작 씬과 화면 전환 | `app/` |
| 공, 플리퍼, 충돌과 플레이 결과 계산 | `pinball/` |
| 보드 구성, 목표와 웨이브 콘텐츠 | `stages/` |
| 보상 생성, 가격과 구매 유효성 | `rewards/` |
| 유물과 패시브 효과 적용 | `relics/` |
| 화면 표시와 사용자 입력 표현 | `ui/` |

`game_flow`는 화면 이름, 씬 경로, 버튼 또는 물리 노드를 도메인 상태로 사용하지 않는다.

## 디렉토리 라우팅

| 경로 | 책임 | 작업 시 추가 확인 |
| --- | --- | --- |
| `run/` | 한 런의 시작, 스테이지 순서와 종료 | `run/AGENTS.md` |
| `stage/` | 한 스테이지의 일반 웨이브·보상·보스·리트라이 순서 | `stage/AGENTS.md` |
| `wave/` | 한 웨이브의 시작, 실행과 단일 종료 결과 | `wave/AGENTS.md` |
| `.work/` | 승인된 작업 계획, 변경 기록과 QA 증거 | 생성하거나 갱신할 때 `WORKFLOW.md` |

## 기준 진행 순서

```text
런 시작
→ 스테이지 시작
→ 일반 웨이브 1 → 보상 구매 완료
→ 일반 웨이브 2 → 보상 구매 완료
→ 일반 웨이브 3 → 보상 구매 완료
→ 보스 웨이브
→ 스테이지 클리어 또는 실패
→ 다음 스테이지, 리트라이 또는 런 종료
```

- 각 단계는 한 번에 하나의 유효한 전환만 수행한다.
- 동일한 완료·실패 결과가 중복 전달돼도 진행이 두 번 증가하지 않아야 한다.
- 유효하지 않은 상태 전환은 명시적으로 거부하고 진단 가능한 결과를 남긴다.
- 웨이브 결과는 공개 결과 계약으로만 받고, 계산 과정이나 물리 노드를 참조하지 않는다.
- 보상 구매의 상세 판단은 `rewards/`에 맡기고 완료 결과만 진행에 반영한다.

## 의존성 방향

```text
app → game_flow
game_flow → stages
game_flow → rewards
game_flow → shared
```

- `game_flow`는 `app/`, `ui/`, `presentation/`을 참조하지 않는다.
- `stages/`와 `rewards/`가 `game_flow/`를 역참조하지 않게 공개 결과 또는 `shared/` 계약으로 연결한다.
- 아직 생성되지 않은 기능은 추측해 임시 구현하지 않고 해당 단계의 승인 후 연결한다.

## 미확정 정책 처리

`GROUND_RULES.md`의 `D-01`~`D-09`는 사용자 결정 전까지 확정하지 않는다. 특히 실패 조건, 목숨, 리트라이 초기화 범위, 코인 유지 범위, 구매 제한과 보상 배치 시점은 코드 상수나 문서 규칙으로 임의 고정하지 않는다. 프로토타입에 임시 값이 필요하면 승인된 작업 기록에 가정, 교체 지점과 검증 항목을 남긴다.

## 필수 검증

- 기준 진행 순서에서 누락, 역행 또는 중복 진입이 없는지 확인한다.
- 세 번째 보상 완료 뒤에만 보스 웨이브로 진입하는지 확인한다.
- 중복 결과 전달이 웨이브나 스테이지를 두 번 진행시키지 않는지 확인한다.
- 스테이지 실패가 이전 스테이지의 완료 상태를 훼손하지 않는지 확인한다.
- 도메인 코드가 화면 이름, 씬 경로와 물리 노드에 의존하지 않는지 확인한다.
- 관련 미확정 정책을 임의로 확정하지 않았는지 확인한다.
