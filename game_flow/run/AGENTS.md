# run 진행 에이전트 규칙

## 책임

- 런 시작, 현재 스테이지와 완료한 스테이지를 관리한다.
- 승인된 스테이지 순서 1→2→3을 진행하고 마지막 완료 뒤 런 완료 결과를 만든다.
- 현재 스테이지의 클리어 결과를 한 번만 반영한다.

스테이지 내부 웨이브 순서, 로비 화면, 저장 데이터와 영구 해금은 소유하지 않는다. 런은 스테이지 씬을 직접 열지 않고 공개 식별자와 결과로만 협력한다.

## 전환 규칙

- 런은 개념적으로 `대기 → 진행 중 → 완료` 순서로만 전환한다.
- 진행 중에는 구성된 현재 스테이지 하나만 활성 상태다.
- 현재 스테이지가 아닌 대상의 완료 결과는 거부한다.
- 동일한 스테이지 완료 결과를 다시 받아도 다음 스테이지를 건너뛰지 않는다.
- 현재 스테이지 리트라이는 이미 완료한 이전 스테이지 상태를 변경하지 않는다.
- 스테이지 사이에 어떤 값을 유지할지는 `D-08`, 런 종료 뒤 영구 보존 범위는 `D-09` 결정 전까지 확정하지 않는다.

## 필수 검증

- 스테이지가 1→2→3 순서로만 진행되는지 확인한다.
- 중복 완료, 잘못된 대상과 완료 후 추가 전환이 거부되는지 확인한다.
- 현재 스테이지 리트라이 후 이전 스테이지 완료 기록이 유지되는지 확인한다.
- 영구 성장이나 저장 정책을 이 기능이 임의로 만들지 않았는지 확인한다.
