# wave 진행 에이전트 규칙

## 책임

- 한 일반 또는 보스 웨이브의 준비, 플레이와 종료 상태를 관리한다.
- 공개 결과 계약으로 받은 클리어 또는 실패 중 하나만 최종 결과로 확정한다.
- 일반 웨이브 결과는 상위 스테이지의 보상 진입, 보스 결과는 스테이지 종료 판단에 전달한다.

보드 생성, 목표 스코어, 핀볼 물리, 보스 계산, UI 표시는 소유하지 않는다. 해당 기능의 노드나 씬을 직접 참조하지 않는다.

## 전환 규칙

- 웨이브는 개념적으로 `준비 → 플레이 중 → 클리어 또는 실패` 순서로만 전환한다.
- 플레이 중 상태에서만 최종 결과를 받을 수 있다.
- 한 번 종료된 웨이브의 추가 결과는 진행을 바꾸지 않고 일관되게 거부한다.
- 일반 웨이브 클리어는 직접 다음 웨이브나 스테이지로 이동하지 않고 상위 스테이지에 결과를 전달한다.
- 보스 웨이브 클리어도 직접 런을 변경하지 않고 상위 스테이지에 결과를 전달한다.
- 실패 조건과 목숨 소진 방식은 `D-01`, `D-02` 결정 전까지 이 기능에서 계산하지 않는다.

## 필수 검증

- 준비 상태의 결과 수신, 종료 뒤 재종료와 잘못된 상태 전환이 거부되는지 확인한다.
- 웨이브 하나에 최종 결과가 정확히 하나만 남는지 확인한다.
- 일반과 보스 결과가 올바른 상위 처리로만 전달되는지 확인한다.
- 화면, 씬, 물리 구현에 대한 직접 의존이 없는지 확인한다.
