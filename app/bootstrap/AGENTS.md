# app/bootstrap 에이전트 규칙

## 상위 규칙

루트 `AGENTS.md`, 루트 `WORKFLOW.md`의 관련 절과 `app/AGENTS.md`를 먼저 읽는다.

## 책임

- 애플리케이션 루트 씬
- 필수 기능 루트의 생성과 초기화 순서
- 초기 화면 진입
- 시작 실패 시 오류 보고와 안전한 중단 또는 복구

## 제외 범위

- 게임 진행 상태와 초기값 결정
- 개별 기능 내부 노드 생성
- 로비·플레이 화면의 UI 구성
- 기능 간 범용 이벤트 버스 구현

## 의존 규칙

- 기능을 생성할 수 있지만 공개된 초기화 계약만 호출한다.
- 기능 내부 노드 경로에 직접 의존하지 않는다.
- `navigation/`을 통해 초기 화면으로 진입한다.
- 오토로드는 기본 선택이 아니며 사용자 승인 없이 추가하지 않는다.

## 검증

- 시작 시 필수 기능이 정확히 한 번 생성되는가
- 초기화 순서가 결정적이고 문서화돼 있는가
- 필수 기능 초기화 실패가 조용히 무시되지 않는가
- 프로젝트를 반복 실행해도 이전 실행 상태에 의존하지 않는가
- `project.godot`의 메인 씬과 실제 루트 씬이 일치하는가
