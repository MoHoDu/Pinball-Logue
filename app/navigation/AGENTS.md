# app/navigation 에이전트 규칙

## 상위 규칙

루트 `AGENTS.md`, 루트 `WORKFLOW.md`의 관련 절과 `app/AGENTS.md`를 먼저 읽는다.

## 책임

- 최상위 화면 경로 정의
- 화면 진입·퇴장과 현재 화면 수명 관리
- 전환 중 중복 요청 차단
- 로딩·전환 실패 시 이전 화면 또는 안전 화면 복귀
- 플랫폼의 뒤로 가기·종료 요청을 앱 정책에 연결

## 제외 범위

- 웨이브 클리어와 실패 조건 판단
- 개별 화면 내부 버튼·레이아웃·애니메이션 구현
- 런과 스테이지 데이터 저장
- 카메라·음향·이펙트 연출 자체 구현

## 전환 규칙

- 한 시점에 활성 최상위 화면은 하나를 기본으로 한다.
- 전환 요청은 명시적인 화면 식별자와 필요한 최소 데이터만 받는다.
- 게임 도메인 객체 전체를 화면 간 전환 인자로 넘기지 않는다.
- 전환 완료 전 동일하거나 충돌하는 요청을 중복 실행하지 않는다.
- 화면 경로 이름은 `GAME_TERMS.md`의 표준 용어를 따른다.

## 검증

- 허용된 화면 경로가 모두 도달 가능한가
- 허용되지 않은 전환이 차단되거나 안전하게 처리되는가
- 같은 전환을 반복 요청해도 화면이 중복 생성되지 않는가
- 이전 화면의 시그널과 타이머가 남지 않는가
- 전환 실패 후 입력 가능한 안전 상태로 복귀하는가
- 로비 → 플레이 → 결과 → 로비 반복 시 메모리와 노드 수가 계속 증가하지 않는가
