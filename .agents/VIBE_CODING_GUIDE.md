# Pinball_Logue 바이브 코딩 사용자 가이드

## 기본 사용 절차

1. 원하는 결과, 변경 이유와 관련 기능을 설명한다. 기획이 바뀌었다면 현재 요청에서 변경 소스도 함께 제공한다.
2. AI가 로컬 문서와 대상 디렉토리를 확인하고 단계, 제외 범위와 검증 항목을 제안한다.
3. 계획을 확인해 `승인`, `수정` 또는 `거부`를 명확히 답한다.
4. 승인 후 AI는 한 단계씩 구현하고 자체 검증 결과와 변경 파일을 보고한다.
5. 결과를 확인해 다음 단계 진행, 수정 또는 되돌아갈 계획을 지시한다.
6. 전체 구현 뒤 전문 QA 목록과 증거를 확인한다.
7. Jira 버그 등록, Confluence 게시, 브랜치·커밋·Push·PR은 필요할 때 각각 승인한다.

## 요청할 때 포함하면 좋은 정보

- 무엇을 바꾸려는지와 플레이어가 느껴야 할 결과
- 현재 문제의 재현 순서와 기대 결과
- 관련 기능 또는 화면
- 바꾸지 않아야 할 범위
- 정확한 기획 변경 소스와 적용 요청
- 원하는 수치, 캡처, 녹화 또는 테스트 조건

## 짧은 요청 예시

```text
보상 카드 구매 후 코인이 두 번 차감되는 문제를 진단해줘.
관련 로컬 문서와 현재 구현을 확인하고 계획과 검증 항목까지만 먼저 보여줘.
```

```text
이번 계획의 1단계만 승인할게. 구현 후 중복 입력 10회 결과와 로그를 보여줘.
```

## 피드백 방법

- 승인: `계획 전체를 승인해`, `1단계만 진행해`
- 수정: `UI 변경은 제외하고 구매 로직만 다시 계획해`
- 거부: `이 방향은 사용하지 않을게. 기존 SPEC을 유지해`
- 검증 추가: `Web 1280×720과 30 FPS 조건도 포함해`

채팅에만 남은 결정은 다른 작업자가 신뢰할 수 없다. 승인된 기획, 계획, QA와 인수인계가 로컬 Markdown에 반영됐는지 최종 보고에서 확인한다.
