# presentation 기능 에이전트 규칙

## 구조

| 경로 | 책임 |
| --- | --- |
| `camera/` | 공 추적, 화면 흔들림과 보스 프레이밍 |
| `vfx/` | 충돌, 콤보, 스코어, 유물과 보스 시각효과 |
| `audio/` | 충돌, UI, 보스와 배경 음향 재생 정책 |
| `feedback/` | 히트 스톱과 시청각 피드백 조합·강도 |

## 경계와 규칙

```text
app → presentation
presentation → shared
```

- 공개 게임 이벤트를 구독하며 물리, 스코어, 보스, 구매와 진행 원본 상태를 변경하지 않는다.
- 공과 경로의 가독성을 이펙트와 카메라 움직임보다 우선한다.
- 같은 이벤트의 중복 재생을 막고 이벤트 식별자로 원인을 추적한다.
- 프레임률과 Web 성능에 따라 단계적으로 품질을 낮출 수 있게 하되 필수 피드백은 유지한다.
- 피드백 강도, 시간과 동시 재생 수는 설정값으로 두고 광과민성·과도한 흔들림을 피한다.

## 필수 검증

- 첫 10초 안의 강한 충돌 또는 연쇄 충돌 원인을 시청각으로 읽을 수 있는지 확인한다.
- 범퍼, 콤보, 코인, 유물과 보스 타격을 서로 구분할 수 있는지 확인한다.
- 최대 속도·최대 이펙트에서도 공과 목표 경로가 추적 가능한지 확인한다.
- 60·30 FPS와 Web 환경에서 피드백 시간, 오디오 중복과 프레임 저하를 비교한다.
