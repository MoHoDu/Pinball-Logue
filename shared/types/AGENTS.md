# shared types 에이전트 규칙

- 이름에 의미와 단위를 드러내고 원시 숫자·문자열의 혼용을 줄인다.
- 값 타입은 가능한 불변으로 유지하고 생성 시 유효 범위를 검사한다.
- 열거형에 알 수 없는 값 처리와 이전 저장 데이터 호환 정책을 둔다.
- 기능별 계산과 상태 전환 메서드를 공용 값 타입에 넣지 않는다.
- 최소·최대·경계값과 직렬화 왕복을 검증한다.
