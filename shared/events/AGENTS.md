# shared events 에이전트 규칙

- 이벤트는 고유 식별자, 종류, 발생 시각 또는 순서, 출처와 불변 페이로드를 가진다.
- 이미 발생한 사실만 표현하며 소비자가 실행할 게임 규칙을 숨기지 않는다.
- 전달 순서, 중복, 지연과 구독 해제 정책을 소비 기능이 검증할 수 있게 한다.
- UI 노드, 장면 경로와 기능 내부 객체 참조를 페이로드에 넣지 않는다.
- 같은 이벤트 재전송, 순서 역전과 구독자 없음 상태를 검증한다.
