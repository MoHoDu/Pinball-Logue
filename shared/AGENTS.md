# shared 기능 에이전트 규칙

## 진입 기준과 구조

| 경로 | 책임 |
| --- | --- |
| `contracts/` | 기능 경계를 넘는 명령·결과 인터페이스 |
| `events/` | 발행자와 소비자를 분리하는 중립 이벤트 봉투 |
| `types/` | 단위가 명확한 불변 값 타입과 열거형 |
| `ids/` | 기능 경계를 넘는 안정적인 식별자 타입 |

- 둘 이상의 기능이 실제로 소비하고 의미가 중립적인 요소만 둔다.
- 단일 기능에서만 쓰거나 미래에 쓸 것 같은 코드는 원래 기능에 유지한다.
- 기능별 게임 규칙, 전역 상태, 만능 유틸리티와 편의용 서비스는 두지 않는다.

## 의존성 규칙

```text
feature → shared
shared → Godot 기본 API만
```

- `shared/`는 `app/`, 도메인 기능, `ui/`, `presentation/`을 참조하지 않는다.
- 계약의 의미와 수용 기준은 소유 기능 문서에 두고 `shared/`에는 전송 형태만 둔다.
- 새 공용 요소 계획에는 최소 두 소비 기능, 소유자, 버전 영향과 제거 조건을 기록한다.
- 기존 계약을 깨는 변경은 모든 소비자와 통합 QA를 같은 승인 범위에 포함한다.

## 필수 검증

- 모든 공용 요소에 실제 소비 기능이 둘 이상 존재하는지 확인한다.
- 단위, null 가능성, 오류와 버전 호환성이 명시됐는지 확인한다.
- 직렬화 왕복, 잘못된 값과 알 수 없는 식별자를 검증한다.
- `shared/`가 기능별 상태나 규칙을 소유하지 않는지 의존성 검사를 실행한다.
