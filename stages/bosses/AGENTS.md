# stage bosses 에이전트 규칙

## 책임

- 보스의 식별자, 충돌 영역, HP·진행도와 완료 조건 참조를 제공한다.
- 스테이지별 대표 기믹 또는 공격 패턴 하나의 게임 규칙을 정의한다.
- 보스 상태와 공격의 준비·위험 범위·발동·결과 이벤트를 제공한다.

## 규칙

- 보스는 공의 직접 타격으로 진행되고 일반 웨이브의 플리퍼 선택과 공 패링을 그대로 사용한다.
- 보스와 플리퍼의 거리, 충돌 영역과 기믹 시간은 설정값으로 노출한다.
- 공격 예고의 게임 상태는 제공하지만 카메라, 이펙트와 사운드를 직접 실행하지 않는다.
- 보스 결과는 스테이지 상태를 직접 바꾸지 않고 공개 결과로 전달한다.
- 공격력 패시브의 보스 피해 적용은 `D-07` 결정 전까지 임의로 확정하지 않는다.

## 필수 검증

- 대표 기믹의 대응법을 한 문장으로 설명할 수 있는지 확인한다.
- 준비→위험 범위→발동→결과 단계와 대응 시간이 구분되는지 확인한다.
- 거리 설정별 직접 타격 성공률, 공 패링 효과와 타격 누락을 비교한다.
- 일반 웨이브에서 획득한 빌드와 익힌 조작이 보스 결과에 영향을 주는지 확인한다.
- 보스 충돌의 중복 피해, 고속 관통과 안전지대 필승 조합을 확인한다.
