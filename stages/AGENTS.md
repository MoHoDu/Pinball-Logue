# stages 기능 에이전트 규칙

## 적용 범위와 필수 문서

이 규칙은 `stages/` 전체에 적용한다. 루트부터 대상까지 모든 `AGENTS.md`를 읽는다. 용어는 `GAME_TERMS.md`, 수용 기준은 `GROUND_RULES.md`의 B·C·G·H를 사용한다. 계획·변경·QA에는 `WORKFLOW.md`를 적용한다.

## 책임

- 스테이지 1·2·3의 식별자와 승인된 콘텐츠 구성을 제공한다.
- 핀볼 보드의 형태, 재사용 물리 요소 위치와 콘텐츠 배치 지점을 정의한다.
- 일반 웨이브의 목표 스코어와 웨이브별 콘텐츠 설정을 제공한다.
- 보스 웨이브의 보스 데이터, 위치와 대표 기믹을 정의한다.
- 보드에서 획득하는 웨이브 아이템의 데이터와 출현 설정을 제공한다.

## 의존성 방향

```text
game_flow → stages
stages → pinball
stages → shared
```

- `game_flow/`는 공개 스테이지·웨이브 설정을 조회하고 진행 상태는 자체 관리한다.
- `stages/`는 `pinball/`의 공개 장면과 설정 계약으로 보드를 구성하며 내부 물리 노드를 변경하지 않는다.
- `stages/`는 `game_flow/`, `app/`, `ui/`, `presentation/`을 참조하지 않는다.
- 보스 타격, 목표 달성, 아이템 획득은 중립 결과로 전달하며 화면 전환을 직접 실행하지 않는다.

## 콘텐츠 불변 규칙

- 한 스테이지는 일반 웨이브 3개와 보스 웨이브 1개를 제공한다.
- 일반 웨이브는 목표 스코어 달성이 핵심이며 현재 프로토타입에 일반 몬스터를 사용하지 않는다.
- 대부분의 주요 충돌이 스코어 진행에 기여하고 작은 단일 표적 때문에 진행이 정체되지 않게 한다.
- 보스는 스테이지당 대표 기믹 또는 공격 패턴 하나를 중심으로 구성한다.
- 일반 웨이브에서 사용한 플리퍼 선택, 공 패링과 빌드가 보스에서도 유효해야 한다.
- 보드 위치, 목표값, 오브젝트 수와 보스 거리는 하드코딩하지 않고 설정 데이터로 관리한다.

## 필수 검증

- 각 스테이지가 일반 웨이브 3개와 보스 웨이브 1개를 정확히 참조하는지 확인한다.
- 모든 참조 식별자가 존재하고 잘못된 보드·목표·보스 조합을 로드 전에 거부하는지 확인한다.
- 첫 10초 안에 강한 충돌 또는 연쇄 충돌이 가능한 배치인지 측정한다.
- 안전·고득점·보스 공격 경로가 시각적으로 구분되고 실제 결과에 차이를 만드는지 확인한다.
- 고속 공의 터널링, 끼임, 무한 반복, 과속과 필승 배치를 확인한다.
- 코인과 안정 보상 경제를 검증할 때 획득량, 배치와 가격을 같은 조건에서 함께 측정한다.
