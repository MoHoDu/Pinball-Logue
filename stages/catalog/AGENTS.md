# stage catalog 에이전트 규칙

## 책임

- 스테이지의 안정적인 식별자, 표시용 메타데이터와 승인된 순서를 제공한다.
- 각 스테이지가 사용할 일반 웨이브 3개와 보스 웨이브 구성의 참조를 묶는다.
- 참조가 완전한지 로드 전에 검증한다.

## 규칙

- 식별자는 표시 이름과 분리하고 저장·진행 데이터에서 임의로 변경하지 않는다.
- 현재 카탈로그는 스테이지 1→2→3을 제공하지만 진입 가능 여부와 현재 위치는 `game_flow/`가 결정한다.
- 카탈로그는 장면을 전환하거나 런 상태를 변경하지 않는다.
- 보드, 목표, 일반 웨이브와 보스 참조를 경로 문자열보다 검증 가능한 리소스 또는 식별자로 연결한다.

## 필수 검증

- 식별자 중복과 누락이 없는지 확인한다.
- 각 스테이지에 일반 웨이브 3개와 보스 웨이브 1개가 연결됐는지 확인한다.
- 삭제·이름 변경 시 참조 끊김을 로드 전에 발견하는지 확인한다.
- 표시 이름 변경이 진행 식별자와 저장 참조를 바꾸지 않는지 확인한다.
