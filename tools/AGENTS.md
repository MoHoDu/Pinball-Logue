# tools 에이전트 규칙

## 구조

| 경로 | 책임 |
| --- | --- |
| `validation/` | 문서·의존성·Godot·Web·데이터 검증 |
| `generation/` | 승인된 스키마 기반 콘텐츠·초안 생성 |
| `capture/` | 로그, 수치, 화면 캡처와 녹화 수집 |
| `reporting/` | QA·일자 기록용 로컬 보고서 구성 |

## 공통 안전 규칙

- 도구 실행 전 대상 기능 `AGENTS.md`, 입력, 출력, 변경 범위와 검증 기준을 확인한다.
- 기본은 읽기 전용 또는 미리보기이며 제품 파일 변경은 사용자가 승인한 작업 단계에서만 수행한다.
- 입력 파일, 시드, 모델·도구 버전, 명령, 생성 결과와 검증 결과를 기록한다.
- 생성물은 승인된 스키마로 검증하고 사람이 검토하기 전 확정 기획이나 제품 데이터로 취급하지 않는다.
- 외부 접근은 루트 정책에 따라 현재 요청에서 사용자가 지정하고 허용한 소스만 사용한다.
- 개인 절대 경로, 인증 정보, 외부 링크와 비밀값을 생성물에 기록하지 않는다.
- 도구 실패가 원본 파일을 부분 변경하지 않도록 임시 출력→검증→원자적 교체 순서를 사용한다.

## 필수 검증

- 같은 입력·시드·버전으로 결과가 재현되는지 확인한다.
- 잘못된 입력, 출력 충돌, 중단과 재실행에서 원본이 보존되는지 확인한다.
- 생성 결과가 대상 기능 스키마, 용어, 그라운드룰과 미확정 결정을 위반하지 않는지 확인한다.
- Godot와 Web 검증 도구는 종료 코드, 로그 경로와 제품 내보내기 파일 목록을 남긴다.
