# ui components 에이전트 규칙

- 두 화면 이상이 실제 사용하는 표현만 이동하고 한 화면 전용 요소는 해당 화면에 둔다.
- 입력, 비활성, 포커스, 로딩, 성공과 오류 상태를 일관되게 제공한다.
- 컴포넌트는 도메인 객체 대신 표시 전용 데이터와 사용자 의도 콜백을 받는다.
- 텍스트 길이 변화, 수치 자릿수, 작은 뷰포트와 키보드 포커스를 검증한다.
- 기능별 게임 규칙이나 전역 상태를 공용 컴포넌트에 숨기지 않는다.
