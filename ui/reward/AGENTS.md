# reward ui 에이전트 규칙

- 화면 명칭은 `보상 선택 화면`, 사용자 행동은 `보상 구매`로 표현한다.
- 카드 3개의 유물 종류·수량, 패시브, 가격과 보유 코인을 동시에 표시한다.
- 무료 1개 선택처럼 보이지 않게 구매 가능·구매 완료·잔액 부족 상태를 구분한다.
- 구매 가능성을 UI에서 독자 계산하지 않고 구매 결과에 따라 잔액과 카드 상태를 갱신한다.
- 서로 다른 카드의 복수 구매를 지원하되 총 상한 표시는 `D-05` 결정을 따른다.
- 연타·동시 구매·실패 후 재시도에서 중복 차감처럼 보이는 표시 오류를 검증한다.
